What's New in Orbit 0.6.0

Orbit is now open source under the MIT License: github.com/simonren/orbit. This release brings new features, a real help system, security hardening, and broad UX polish.

New features

  • Launch at startup — opt in via Settings → General, or check the box on the first-run welcome sheet.
  • Optional helper / non-root mode — you can decline the privileged helper and manage lo0 aliases yourself with sudo ifconfig. Orbit prompts with a one-click "Install Helper Now" or shows the manual command on bind failure.
  • First-run welcome sheet — quick yes/no on launch-at-login and helper install, with clear what-each-does context. Existing users see no prompt on update; their setup is preserved.
  • Settings window (⌘,) — manage launch-at-login, helper install/uninstall, and orb-kubectl install/uninstall in one place.
  • Orbit Help window — accessible via Help → Orbit Help (⌘?). Twelve articles covering environments, services, K8s import, the privileged helper, network modes, settings, and a troubleshooting playbook. Fully offline; searchable.

orb-kubectl trust improvements

  • SHA-256 of the downloaded archive is verified against a value pinned in Orbit before installation. A mismatch aborts; the on-disk binary is not replaced.
  • Install dialog discloses the source fork (github.com/simonren/kubernetes, branch feature/resilient-port-forward), the exact patch path, the expected checksum, and install location — with an explicit "decline and use plain kubectl" alternative.
  • Management moved into Settings → Tools (single source of truth).
  • If orb-kubectl isn't installed, the Kubernetes Import sheet now defaults to plain kubectl from $PATH (previously it could get stuck on orb-kubectl with the picker disabled).

Security & robustness

  • Privileged helper v1.3.0 — adds a self-destruct RPC so Uninstall actually removes the on-disk plist and binary (previously left as inert root-owned files). Also tightens process-group cleanup: the helper now validates that any PGID it would killpg is a descendant of the registered app PID.
  • Import safety — service commands are now displayed in the import preview (single + bulk), and imported services land disabled by default. A shared .orbit.json can no longer auto-run attacker-supplied shell when toggled.
  • PATH hardening — Orbit's user-writable bin directory is now appended to $PATH for spawned services, not prepended. System binaries (bash, kubectl, etc.) always resolve from system paths first.
  • Apache-2.0 attribution for orb-kubectl in NOTICE; source fork referenced in the in-app trust disclosure.

UX polish

  • Menubar popover switched to the native rounded-rectangle MenuBarExtra style with system frosted material (matches Tailscale, Just Focus, etc.). The triangular pointer is gone.
  • "Copy for AI" button in the environment detail header is now a labeled button with a green ✓ confirmation when clicked, and the header buttons are reordered for clearer intent.
  • Bumped small text in sheets from 10 pt to 12 pt for readability.
  • Esc closes the log window; main window no longer beeps on Esc.
  • "Show Window" in the menubar popup now actually dismisses the popup.
  • Toggling an environment that needs an admin action from the menubar now opens the main window so the prompt is visible.
One-time admin prompt on upgrade. When you first activate an environment after updating, Orbit will ask permission to upgrade its privileged helper from v1.2.0 to v1.3.0. This is the helper-side change for the cleaner uninstall and tighter process-group validation above. Approve once and the prompt won't appear again.
